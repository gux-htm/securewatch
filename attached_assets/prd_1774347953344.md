# AegisGuard – System Architecture & Design

## High-Level Components
1. **Central Server** – FastAPI + PostgreSQL + Redis + OpenVPN + IPS/IDS engine
2. **Client Agent** – Lightweight Python daemon (watchdog + crypto + reporting)
3. **VPN Layer** – Managed OpenVPN instances (one per virtual network)
4. **Crypto Authority** – Internal CA using OpenSSL/cryptography
5. **Dashboard** – FastAPI + HTMX web interface
6. **Audit Store** – PostgreSQL with signed JSON blobs

## Data Flow
- Client Agent → WebSocket (TLS) → Server (event + signature + hash)
- Admin → Dashboard → Server → Push policy to agents or OpenVPN
- Packet traffic → Scapy/nfq → IPS/IDS → Firewall decision
- VPN connection request → Certificate + MAC + IP verification → Allow/Deny

## Database Schema (key tables)
- devices (id, mac, ip, cert_fingerprint, network_id, last_seen)
- networks (id, name, vpn_config)
- policies (resource_path, user_id, permissions: view/edit/delete/rename)
- audit_logs (event_id, device_id, user_cert, file_hash_before, file_hash_after, action, timestamp, signature)
- firewall_rules (rule_id, network_id, action, source, dest, port, protocol)

## Security Design
- Every message signed and verified.
- Private keys never leave their respective device/agent.
- Server master key only used for log sealing.

## Deployment View
- Docker Compose: server, db, redis, openvpn
- Client installer packages (DEB, RPM, MSI, PKG)

This architecture ensures every requirement in the SRS is met.
